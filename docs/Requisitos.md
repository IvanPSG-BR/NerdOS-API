# Requisitos

## Funcionais

- CRUD de Usuário
- Autenticação de Usuário
- CRUD de Clientes
- CRUD de Aparelhos (com vínculo a Cliente)
- CRUD de Ordem de Serviço (OS)
- Gestão de Status da OS
- Consulta de Histórico do Cliente (Aparelhos + OSs)
- Endpoints de Dashboard (Estatísticas de OS e Faturamento)

## Não Funcionais

- Performance (Fastify)
- Type-safety ponta-a-ponta (TypeScript, Prisma, Zod)
- Documentação de API automática (Swagger/OpenAPI via Fastify/Zod)
- Código limpo, modular e não hard-coded para o cliente piloto.
